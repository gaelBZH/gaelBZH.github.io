Lecture 4 : MPI

Message Passing Interface — Résumé complet en français

1. Introduction au calcul haute performance et à MPI

MPI (Message Passing Interface) est une norme de communication standard utilisée en calcul haute performance (HPC). Elle repose sur le paradigme envoi-réception : les processus parallèles s'échangent des données explicitement par des messages.

Principes fondamentaux

  • Envoi d'un message : send(cible, id, données)
  • Réception d'un message : receive(source, id, données)
  • Versatilité : applicable à des architectures très variées (clusters, grilles, supercalculateurs).
  • Support des programmes MPMD (Multiple Program, Multiple Data) et SPMD (Single Program, Multiple Data).

2. Concepts de base de MPI

Communicateur

Un communicateur regroupe un ensemble de processus pouvant communiquer. Le communicateur prédéfini MPI_COMM_WORLD regroupe tous les processus.

Rang d'un processus

Chaque processus possède un rang unique. Pour \(p\) processus :

\[ 0 \;\leq\; \text{rang} \;<\; p \]

Procédures de base

FonctionRôle
MPI_Init(&argc, &argv)Initialise l'environnement MPI
MPI_Comm_size(comm, &size)Retourne le nombre de processus
MPI_Comm_rank(comm, &rank)Retourne le rang du processus appelant
MPI_Finalize()Finalise et libère les ressources MPI

Exemple minimal

#include "mpi.h"
int main(int argc, char** argv) {
    int rank, size;
    MPI_Init(&argc, &argv);
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    printf("Mon numéro %d, dans un groupe de %d processus\n", rank, size);
    MPI_Finalize();
    return 0;
}

3. Communication point-à-point

La communication point-à-point implique exactement deux processus : un émetteur et un récepteur.

Garanties offertes par MPI

  • Progression garantie après initialisation correcte d'une paire envoi-réception.
  • Ordre de réception garanti pour messages de la même source avec le même tag.
  • Équité non garantie pour messages de sources différentes.

Communications bloquantes

  • MPI_Send(buf, count, dtype, dest, tag, comm)
  • MPI_Recv(buf, count, dtype, src, tag, comm, &status)

Les constantes MPI_ANY_SOURCE et MPI_ANY_TAG permettent de recevoir de n'importe quelle source.

Exemple : envoi des rangs vers le processus 0

if (rank != 0) {
    MPI_Send(&rank, 1, MPI_INT, 0, 0, MPI_COMM_WORLD);
} else {
    for (int i = 1; i < size; i++) {
        MPI_Recv(&ranksent, 1, MPI_INT, MPI_ANY_SOURCE,
                 MPI_ANY_TAG, MPI_COMM_WORLD, &status);
        printf("Données du processus : %d (%d)\n",
               ranksent, status.MPI_SOURCE);
    }
}

Communications non-bloquantes

  • MPI_Isend(buf, count, dtype, dest, tag, comm, &req)
  • MPI_Irecv(buf, count, dtype, src, tag, comm, &req)

Rend la main immédiatement, permettant de recouvrir calculs et communications. Nécessite MPI_Wait ou MPI_Test pour vérifier la complétion.

FonctionRôle
MPI_Wait(&req, &stat)Attend la complétion de l'opération
MPI_Test(&req, &flag, &stat)Teste si l'opération est terminée
MPI_Waitall / MPI_TestallAttend/teste toutes les opérations d'un tableau

4. Communications collectives (de groupe)

Les opérations collectives impliquent plus de deux processus simultanément. Elles sont déclenchées en appelant la même procédure sur tous les processus. Toutes sont bloquantes.

Barrière — MPI_Barrier(comm)

Bloque jusqu'à ce que tous les processus l'aient appelée.

Diffusion — MPI_Bcast

Le processus root envoie ses données à tous : \( P_0 \xrightarrow{\text{broadcast}} P_0, P_1, \ldots, P_{p-1} \)

MPI_Bcast(buff, count, datatype, root, comm)

Rassemblement — MPI_Gather

Chaque processus envoie ses données locales au processus root.

MPI_Gather(sbuf, scount, sdtype, rbuf, rcount, rdtype, root, comm)

Dispersion — MPI_Scatter

Le processus root distribue des portions distinctes à chaque processus (inverse de Gather).

Rassemblement global — MPI_Allgather

Chaque processus envoie ses données à tous les autres (Gather + Bcast intégré).

Échange total — MPI_Alltoall

Chaque processus \(i\) envoie le \(j\)-ième élément au processus \(j\) pour tout \(j\).

5. Opérations de réduction

Combine les données de tous les processus via une opération et place le résultat chez root :

MPI_Reduce(sbuf, rbuf, count, datatype, op, root, comm)

MPI_Allreduce diffuse le résultat à tous les processus (équivalent Reduce + Bcast) :

6. Exemple complet : intégration numérique parallèle

Calcul de \(\displaystyle I = \int_a^b f(x)\,dx\) par la méthode des trapèzes avec \(N\) intervalles répartis sur \(p\) processus.

\[ I \approx \sum_{i=0}^{N-1} \frac{f(x_i) + f(x_{i+1})}{2} \Delta x, \quad \Delta x = \frac{b-a}{N} \]

Étapes

  1. Processus 0 lit \(a\), \(b\), \(N\).
  2. Diffusion via MPI_Bcast.
  3. Chaque processus calcule sa contribution locale (\(n_{\text{loc}} = \lceil N/p \rceil\) intervalles).
  4. Réduction avec MPI_Reduce(..., MPI_SUM, ...).
n_loc = ceil(N / size);
dx    = (b - a) / N;
x1    = a + rank * n_loc * dx;
if (rank == size - 1) n_loc = N - n_loc * (size - 1);
c = 0;
for (i = 0; i < n_loc; i++) {
    x2 = x1 + dx;
    c += 0.5 * (f(x1) + f(x2)) * dx;
    x1 = x2;
}
MPI_Reduce(&c, &integral, 1, MPI_DOUBLE, MPI_SUM, root, MPI_COMM_WORLD);

Exercices

Exercice 1 — Initialisation et identification

Écrivez un programme MPI en C qui fait afficher à chaque processus : "Je suis le processus \(r\) parmi \(p\)". Quelles fonctions MPI sont indispensables ? Pour \(p = 8\), quels rangs sont possibles ?

Indice
Utilisez MPI_Init, MPI_Comm_rank, MPI_Comm_size, MPI_Finalize. Les rangs vont de \(0\) à \(7\).
Exercice 2 — Communication point-à-point bloquante

Avec \(p\) processus, chaque processus \(r \neq 0\) envoie son rang au processus 0. Celui-ci affiche leur somme. Exprimez cette somme mathématiquement et vérifiez pour \(p = 5\).

Indice
\(S = \sum_{r=1}^{p-1} r = \frac{(p-1)p}{2}\). Pour \(p=5\) : \(S = 10\).
Exercice 3 — Diffusion et réduction : calcul de \(\pi\)

Approximez \(\pi\) en parallèle par la méthode des rectangles : \(\pi = \int_0^1 \frac{4}{1+x^2}\,dx\). Le processus 0 lit \(N\). Utilisez MPI_Bcast puis MPI_Reduce avec MPI_SUM.

Indice
Contribution locale du processus \(r\) : \(\displaystyle c = \sum_{i \,:\, i \bmod p = r} \frac{4}{1 + \left(\frac{i+0{,}5}{N}\right)^2} \cdot \frac{1}{N}\).
Exercice 4 — Communication non-bloquante

Reprenez l'exercice 2 en remplaçant MPI_Send par MPI_Isend. Expliquez pourquoi il faut appeler MPI_Wait après l'envoi non-bloquant.

Indice
Le buffer ne peut être réutilisé qu'après que MPI_Wait confirme la complétion — sinon les données peuvent être corrompues.
Exercice 5 — Scatter et Gather : normalisation d'un vecteur

Le processus 0 dispose de \(\mathbf{v} \in \mathbb{R}^N\). Décrivez comment : (1) distribuer les \(\frac{N}{p}\) composantes via MPI_Scatter, (2) calculer \(\|\mathbf{v}\|^2\) via MPI_Reduce, (3) normaliser localement après MPI_Bcast, (4) rassembler via MPI_Gather.

Indice
\(\|\mathbf{v}\| = \sqrt{\sum_r s_r}\) où \(s_r = \sum_i v_i^2\) est la contribution locale. Après normalisation : \(v_i \leftarrow \frac{v_i}{\|\mathbf{v}\|}\).
Exercice 6 — Allreduce : moyenne globale

Chaque processus \(r\) génère \(x_r\). Calculez \(\bar{x} = \frac{1}{p}\sum_{r=0}^{p-1} x_r\) en utilisant MPI_Allreduce. Expliquez l'avantage par rapport à MPI_Reduce + MPI_Bcast.

Indice
MPI_Allreduce combine les deux opérations en une seule, plus efficace car elle utilise un algorithme de réduction en arbre optimisé.
Exercice 7 — Alltoall : transposition de matrice distribuée

Avec \(p\) processus, le processus \(r\) possède la ligne \(r\) d'une matrice \(p \times p\) : \([A_{r,0}, A_{r,1}, \ldots, A_{r,p-1}]\). Après MPI_Alltoall, que contient le buffer du processus \(r\) ? Reliez ce résultat à la transposition \(A^T\).

Indice
Le processus \(r\) reçoit \(A_{i,r}\) pour tout \(i\) — c'est la colonne \(r\) de \(A\), soit la ligne \(r\) de \(A^T\).
Exercice 8 — Intégration numérique parallèle (d'après le cours)

Calculez \(\displaystyle I = \int_0^{\pi} \sin(x)\,dx\) par la méthode des trapèzes avec \(N = 1000\) et \(p = 4\).

  1. Valeur exacte de \(I\) ?
  2. Nombre d'intervalles par processus (\(n_{\text{loc}} = \lceil N/p \rceil\)) ?
  3. Formule de la contribution locale du processus \(r = 1\) ?
  4. Quelle fonction MPI pour sommer les contributions vers le processus 0 ?
Indice
\(I = [-\cos x]_0^{\pi} = 2\). \(n_{\text{loc}} = 250\). Processus 1 : \(x_1^{(1)} = \frac{\pi}{4}\), contribution \(\displaystyle c_1 = \sum_{i=0}^{249} \frac{\sin(x_i)+\sin(x_i+\Delta x)}{2}\Delta x\). Utiliser MPI_Reduce(..., MPI_SUM, 0, ...).
