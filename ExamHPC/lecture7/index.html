Lecture 7 : Optimisation

Résumé complet en français — High Performance Computing

1. Introduction à l'optimisation des programmes séquentiels

L'optimisation désigne l'ensemble des modifications apportées au code source, ou lors de la traduction du code source en langage machine, dans le but d'obtenir les caractéristiques souhaitées du programme résultant (vitesse d'exécution, taille en mémoire, consommation énergétique, etc.).

Deux facteurs principaux motivant l'optimisation

  • La taille du code : réduire l'empreinte mémoire du binaire produit.
  • La vitesse du code (performance) : accélérer le temps d'exécution.

Comment obtenir de meilleures performances ?

L'amélioration des performances est le plus souvent atteinte par :

  • La réduction du nombre d'opérations effectuées.
  • L'optimisation des accès mémoire (localité spatiale et temporelle).
  • L'exploitation efficace du pipeline du processeur, notamment par la suppression des dépendances entre instructions.

Méthodes d'optimisation disponibles

  • Manuellement, en appliquant des techniques classiques directement dans le code source.
  • Via les options d'optimisation du compilateur (par exemple les niveaux -O1, -O2, -O3 de GCC).
  • En déléguant certaines parties du calcul à des bibliothèques optimisées (BLAS, LAPACK, Intel MKL, etc.).

Dans les cas favorables, l'optimisation d'un programme séquentiel peut réduire le temps d'exécution de plusieurs ordres de grandeur.

2. Techniques d'optimisation sur les variables et les expressions

Ces techniques classiques sont réalisables manuellement et sont également implémentées par les compilateurs optimisants.

Évaluation des constantes (Constant Folding)

Le compilateur calcule à la compilation les expressions dont toutes les opérandes sont des constantes connues.

i = 320 * 200 * 32;   // calculé une seule fois à la compilation

Propagation de copies (Copy Propagation)

Lorsqu'une variable est simplement une copie d'une autre, on substitue directement la variable d'origine :

y = x;
z = 3 + y;    =>    z = 3 + x;   // y est éliminée

Réduction de puissance (Strength Reduction)

On remplace une opération coûteuse par une opération équivalente moins coûteuse :

y = x * 3;    =>    y = x + x + x;

Plus généralement, une multiplication par une puissance de 2 peut être remplacée par un décalage de bits : \( x \times 4 \;\equiv\; x \ll 2 \).

Renommage de variables (Variable Renaming)

Raccourcir le nom d'une variable très utilisée pour améliorer la lisibilité et parfois aider le compilateur :

variable = 4;    =>    v = 4;

Élimination des sous-expressions communes (Common Subexpression Elimination — CSE)

Lorsqu'une même sous-expression apparaît plusieurs fois, on la calcule une seule fois et on stocke le résultat dans une variable temporaire :

a = b * c + g;          tmp = b * c;
d = b * c * e;    =>    a = tmp + g;
                        d = tmp * e;

Ici, le produit \( b \times c \) n'est calculé qu'une seule fois au lieu de deux.

3. Techniques d'optimisation des boucles

Les boucles sont le cœur des programmes intensifs en calcul. Leur optimisation a un impact majeur sur les performances globales.

Simplification des variables d'induction (Induction Variable Simplification)

À chaque itération d'une boucle, certaines variables évoluent de façon prévisible (linéaire). On peut simplifier leur mise à jour en évitant des recalculs inutiles.

Sortie de code invariant de boucle (Loop Invariant Code Motion)

Une expression dont la valeur ne change pas d'une itération à l'autre est calculée hors de la boucle :

// Avant
for (int i = 0; i < n; i++) {
    x = y + z;
    a[i] = 6 * i + x * x;
}

// Après : x et x*x sont invariants
x = y + z;
t1 = x * x;
for (int i = 0; i < n; i++) {
    a[i] = 6 * i + t1;
}

Permutation de boucles (Loop Interchange)

On échange l'ordre des boucles imbriquées pour améliorer la localité des accès mémoire :

// Avant (accès colonnes)
for i from 0 to 10
  for j from 0 to 20
    a[i,j] = i + j

// Après (accès lignes, plus efficace)
for j from 0 to 20
  for i from 0 to 10
    a[i,j] = i + j

Fusion de boucles (Loop Fusion)

On regroupe plusieurs boucles ayant le même domaine d'itération en une seule :

// Avant : deux boucles
for (i = 0; i < 100; i++) a[i] = 1;
for (i = 0; i < 100; i++) b[i] = 2;

// Après : une seule boucle
for (i = 0; i < 100; i++) {
    a[i] = 1;
    b[i] = 2;
}

Fission de boucle (Loop Fission)

Inversement, on peut scinder une boucle en plusieurs boucles plus petites pour améliorer la localité des données ou faciliter la vectorisation.

Déroulage de boucle (Loop Unrolling)

On répète le corps de la boucle plusieurs fois par itération, réduisant le nombre de tests de condition et d'incréments :

// Avant
for (x = 0; x < 100; x++) {
    delete(x);
}

// Après (déroulage par 2)
for (x = 0; x < 100; x += 2) {
    delete(x);
    delete(x+1);
}

Si la boucle effectue \( N \) itérations et qu'on déroule par un facteur \( k \), le nombre de tests de condition passe de \( N \) à \(\frac{N}{k}\).

Découpage en blocs (Blocking / Tiling)

On divise l'espace d'itération en blocs de taille \( B \) pour que les données d'un bloc tiennent dans le cache :

// Avant
for (i = 0; i < N; ++i) { ... }

// Après (blocking avec taille de bloc B)
for (j = 0; j < N; j += B)
  for (i = j; i < min(N, j+B); ++i) { ... }

Cette technique est particulièrement efficace pour les opérations sur des matrices, comme la multiplication matricielle.

4. Optimisation au niveau des instructions

Suppression du code mort (Dead Code Removal)

Le compilateur détecte et supprime les instructions dont le résultat n'est jamais utilisé, ainsi que le code inaccessible :

int a = 24;
int b = 25;   /* Affectation à une variable morte */
int c;
c = a << 2;
return c;
b = 24;       /* Code inaccessible */
return 0;

Ici, b = 25 est mort car b n'est jamais lu, et b = 24; return 0; est inaccessible après return c;.

Élimination de la récursion terminale (Tail-Recursion Elimination)

Lorsqu'un appel récursif est la dernière opération d'une fonction (tail call), on peut le remplacer par un saut simple :

// Fonction avec appel terminal
function foo(data) {
    a(data);
    return b(data);   // tail call -> transformé en saut
}

Inlining de fonctions (Function Inlining)

On remplace un appel de fonction par le corps de cette fonction, supprimant ainsi le coût du mécanisme d'appel :

// Définition inline
inline int max(int a, int b) {
    return (a > b) ? a : b;
}
a = max(x, y);    =>    a = (x > y) ? x : y;

Préchargement logiciel (Software Prefetching)

On insère des instructions de préchargement pour demander au processeur de charger des données en cache avant qu'elles soient nécessaires, masquant ainsi la latence mémoire :

for (i = 0; i < N; i++) {
    fetch(&a[i+1]);   // précharge a[i+1] pour l'itération suivante
    fetch(&b[i+1]);
    sum += a[i] * b[i];
}

Pipeline logiciel (Software Pipelining)

Technique consistant à réordonner les instructions de plusieurs itérations de boucle pour que des instructions de différentes itérations s'exécutent en parallèle dans le pipeline du processeur, évitant les bulles de pipeline dues aux dépendances.

5. Compilateurs optimisants

L'optimisation par le compilateur se déroule à la compilation, généralement après l'analyse syntaxique (parsing) et avant la génération du code objet. Le compilateur opère sur une représentation intermédiaire du programme telle que des arbres syntaxiques, du code à trois adresses, etc.

Analyse du programme par le compilateur

Le parsing permet d'établir la structure du programme en analysant :

  • Le flot de contrôle (control flow) : la possibilité de transférer le contrôle d'un point du code à un autre (branchements, sauts).
  • Le flot de données (data flow) : la détermination des endroits où une variable est définie et des endroits où cette valeur est utilisée.

Bloc de base (Basic Block)

Un bloc de base est une séquence maximale d'instructions telle que :

  • Si une instruction du bloc est exécutée, toutes les autres le sont aussi.
  • On ne peut pas sauter vers l'intérieur du bloc.
  • On ne peut pas sortir du milieu du bloc (une instruction de saut est la dernière).

Exemple de code intermédiaire

while (j < n) {
    k = k + 2j;
    m = 2j;
    j++;
}

traduit en code à trois adresses :

A:
  t1 := j
  t2 := n
  t3 := t1 < t2
  jmp (B) t3      // si j < n, aller en B
  jmp (C)
B:
  t4 := k
  t5 := j
  t6 := t5 * 2
  t7 := t4 + t6
  k  := t7        // k = k + 2j
  t8 := j
  t9 := t8 * 2
  m  := t9        // m = 2j
  jmp (A)

6. Assembleur x86-64 / AMD64

L'assembleur est le niveau le plus bas de représentation du code. Comprendre les instructions assembleur est essentiel pour analyser et optimiser finement le code généré.

Catégories d'instructions x86-64

CatégorieInstructionsDescription
Transfert de données mov, push, pop, ld, st Déplacement de données entre registres et mémoire
Arithmétique et logique add, sub, mul, inc, dec, xor, and, or Opérations standard sur les entiers et les bits
Calcul d'adresse leal Calcul d'adresse effective sans transfert mémoire
Comparaison cmp Évaluation d'une expression conditionnelle (positionne les flags)
Contrôle de flot jmp, jge, je, jl, call, ret Sauts conditionnels/inconditionnels, appels de procédures

Calcul d'adresse mémoire

En x86-64, l'adresse effective d'un opérande mémoire est calculée selon la formule :

\[ \text{adresse} = \text{base} + \text{index} \times \text{scale} + \text{disp} \]
  • base et index : contenus de registres 64 bits.
  • scale : facteur multiplicatif, valeur parmi \(\{1, 2, 4, 8\}\).
  • disp : déplacement immédiat (constante).

Notation AT&T : disp(base, index, scale)

Registres disponibles en x86-64

TailleRegistres principauxRôle notable
64 bits%rax, %rbx, %rcx, …%rbp (frame pointer), %rsp (stack pointer)
32 bits%eax, %ebx, %ecx, …%ebp, %esp
16 bits%ax, %bx, …Moitié basse de %eax, etc.
8 bits%ah, %al, …Octets haut/bas de %ax
SSE (128 bits)%xmm0 – %xmm15Opérations SIMD vectorielles
AVX (256 bits)%ymm0 – %ymm15SIMD étendu (AVX/AVX2)

Syntaxe AT&T (GNU Assembler)

  • Le nom de la commande contient en dernière lettre le type d'argument (ex. : movl pour 32 bits, movq pour 64 bits).
  • La source précède la destination : movl $5, %eax place 5 dans %eax.
  • Les arguments immédiats sont précédés du symbole $.
  • Les noms de registres sont précédés du signe %.
  • Les instructions du coprocesseur en virgule flottante commencent par la lettre f.

Exercices

Exercice 1 — Constant Folding et Strength Reduction

Appliquez les techniques de constant folding et de strength reduction aux expressions suivantes :

  1. result = 1024 * 768 * 4;
  2. y = x * 8;
  3. z = x * 2 + x * 2;
Indice
Pour (1) : le compilateur peut calculer \( 1024 \times 768 \times 4 = 3\,145\,728 \) à la compilation. Pour (2) : multiplier par une puissance de 2 est équivalent à un décalage de bits : \( x \times 8 = x \ll 3 \). Pour (3) : utilisez d'abord la CSE puis la strength reduction.
Exercice 2 — Élimination des sous-expressions communes (CSE)

Appliquez la technique CSE au code suivant :

a = x * y + z;
b = x * y - z;
c = (x * y) * (x * y);

Combien de multiplications sont économisées après optimisation ?

Indice
Introduisez tmp = x * y; avant les trois lignes. Après optimisation, \( x \times y \) est calculé une seule fois. On passe de 4 multiplications à 2.
Exercice 3 — Sortie du code invariant de boucle

Identifiez et sortez les expressions invariantes du code suivant :

int n = 100;
double r = 3.14159;
for (int i = 0; i < n; i++) {
    double aire = r * r * 3.14159;
    result[i] = aire * i;
}

Réécrivez la boucle optimisée et expliquez combien de multiplications sont économisées pour \( n = 100 \).

Indice
aire ne dépend pas de i : on calcule aire = r * r * 3.14159; une seule fois avant la boucle. On économise \( 2 \times (n - 1) = 198 \) multiplications pour \( n = 100 \).
Exercice 4 — Déroulage de boucle

Déroulez la boucle suivante par un facteur 4, puis calculez le nombre de tests de condition avant et après pour \( N = 1000 \) :

for (int i = 0; i < N; i++) {
    somme += tableau[i];
}
Indice
Avant déroulage : \( N \) tests. Après déroulage par \( k \) : \(\frac{N}{k}\) tests. Pour \( N = 1000 \) et \( k = 4 \) : on passe de 1000 à 250 tests.
Exercice 5 — Permutation de boucles et localité mémoire

Soit une matrice \( A \) de taille \( n \times n \) stockée en ordre ligne-majeur. Analysez les deux versions :

// Version 1
for (int i = 0; i < n; i++)
  for (int j = 0; j < n; j++)
    sum += A[i][j];

// Version 2
for (int j = 0; j < n; j++)
  for (int i = 0; i < n; i++)
    sum += A[i][j];

Laquelle est plus efficace en termes d'accès cache ? Justifiez par la formule d'adressage : \(\text{base} + (i \times n + j) \times \text{sizeof(type)}\).

Indice
En version 1, j varie rapidement : les accès sont consécutifs en mémoire (localité spatiale excellente). En version 2, i varie rapidement : les accès sautent de \( n \times \text{sizeof(type)} \) octets, provoquant de nombreux cache misses. La version 1 est préférable.
Exercice 6 — Blocs de base et code intermédiaire

Soit le programme suivant :

int i = 0;
while (i < 10) {
    a[i] = i * 2;
    i++;
}
  1. Traduisez ce code en code intermédiaire à trois adresses.
  2. Identifiez les blocs de base dans le code intermédiaire obtenu.
  3. Appliquez la simplification de variable d'induction pour éliminer la multiplication.
Indice
La variable d'induction est i ; son double t = i * 2 peut être maintenu par simple addition t += 2 à chaque itération (strength reduction sur variable d'induction).
Exercice 7 — Inlining et élimination de code mort

Analysez le code suivant et appliquez successivement l'inlining puis l'élimination du code mort :

inline int cube(int x) {
    return x * x * x;
}

int calcul(int n) {
    int a = 5;
    int b = 10;   // jamais utilisé après
    int r = cube(n);
    int s = cube(2);   // constante
    return r;
    int t = a + r;     // code inaccessible
}

Écrivez la version optimisée finale de calcul.

Indice
Après inlining : cube(n) devient n*n*n et cube(2) devient 8. Ensuite : b est mort, s = 8 est mort, t = a + r est inaccessible. La version finale est : int calcul(int n) { return n * n * n; }
Exercice 8 — Options du compilateur GCC

Pour chacune des situations suivantes, indiquez quelle option GCC est la plus appropriée :

  1. Vous souhaitez que le compilateur remplace automatiquement y = x * 3 par y = x + x + x dans les boucles.
  2. Vous avez une boucle for simple et souhaitez que le compilateur la déroule automatiquement.
  3. Vous voulez que les petites fonctions soient intégrées directement à leurs sites d'appel.
  4. Vous travaillez sur un programme de calcul scientifique et acceptez une légère perte de précision IEEE 754 pour gagner en vitesse.
  5. Vous voulez activer la vectorisation automatique et la parallélisation (niveau d'optimisation le plus élevé).
Indice
(1) -fstrength-reduce ; (2) -funroll-loops ; (3) -finline-functions ; (4) -ffast-math ; (5) -O3 (complétable avec -fopenmp pour le parallélisme).
